👨‍💻 Object-Oriented Programming (OOP) in Python

This page demonstrates professional, real-world OOP concepts like inheritance, encapsulation, abstraction, polymorphism, and special methods — all essential for modern Python development in areas such as backend systems, e-commerce platforms, and financial tools.

1️⃣ User and Admin Access System
class User:
    def __init__(self, username, email):
        self.username = username
        self.email = email

    def login(self):
        return f"{self.username} has logged in."

class Admin(User):
    def __init__(self, username, email, privileges):
        super().__init__(username, email)
        self.privileges = privileges

    def show_privileges(self):
        return f"Admin Privileges: {', '.join(self.privileges)}"

admin = Admin("admin01", "admin@example.com", ["add_user", "delete_user"])
print(admin.login())
print(admin.show_privileges())
  • 🔹 Function Used: `super()`, inheritance
  • 📘 Learned: Role-based extension using subclasses
  • ♻️ Reusability: Login systems, user roles
  • 💬 Comments: Demonstrates base-to-derived relationship
2️⃣ Bank Account with Encapsulation
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.__balance = balance

    def deposit(self, amount):
        self.__balance += amount

    def withdraw(self, amount):
        if amount > self.__balance:
            return "Insufficient funds"
        self.__balance -= amount
        return f"Withdrawn {amount}"

    def get_balance(self):
        return self.__balance

acc = BankAccount("Sumon", 1000)
acc.deposit(500)
print(acc.get_balance())
  • 🔹 Function Used: Private attribute `__balance`
  • 📘 Learned: Protecting data using encapsulation
  • ♻️ Reusability: Secure financial models
  • 💬 Comments: Access restricted with getter methods
3️⃣ Employee and Manager Hierarchy
class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

    def get_details(self):
        return f"{self.name}: ${self.salary}"

class Manager(Employee):
    def __init__(self, name, salary, department):
        super().__init__(name, salary)
        self.department = department

    def get_details(self):
        return f"{self.name} (Manager of {self.department}): ${self.salary}"

mgr = Manager("Sara", 85000, "Engineering")
print(mgr.get_details())
  • 🔹 Function Used: Method overriding
  • 📘 Learned: Customizing behavior in subclasses
  • ♻️ Reusability: Used in HR, org chart logic
  • 💬 Comments: Demonstrates department-specific extension
4️⃣ E-Commerce Product Inheritance
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

    def display(self):
        return f"{self.name} - ${self.price}"

class DigitalProduct(Product):
    def __init__(self, name, price, file_size):
        super().__init__(name, price)
        self.file_size = file_size

dp = DigitalProduct("E-book", 12.99, "5MB")
print(dp.display())
  • 🔹 Function Used: Inheritance
  • 📘 Learned: Sharing common methods in subclasses
  • ♻️ Reusability: E-commerce backend structure
  • 💬 Comments: Allows categorizing different product types
5️⃣ Vehicle Class with Polymorphism
class Vehicle:
    def move(self):
        return "Moving..."

class Car(Vehicle):
    def move(self):
        return "Driving on the road."

class Boat(Vehicle):
    def move(self):
        return "Sailing on water."

for v in [Car(), Boat()]:
    print(v.move())
  • 🔹 Function Used: Polymorphism
  • 📘 Learned: Same method, different behavior
  • ♻️ Reusability: Transport and vehicle simulations
  • 💬 Comments: Common interface with specialized logic
6️⃣ Library System with __str__ and __len__
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __str__(self):
        return f"📖 {self.title}"

    def __len__(self):
        return self.pages

b = Book("Python Mastery", 450)
print(str(b))
print(len(b))
  • 🔹 Function Used: `__str__`, `__len__`
  • 📘 Learned: How to customize object representation
  • ♻️ Reusability: Ideal for any printable or measurable object
  • 💬 Comments: Makes class more readable and iterable
7️⃣ Abstract Base Class for Payment
from abc import ABC, abstractmethod

class PaymentProcessor(ABC):
    @abstractmethod
    def process(self, amount):
        pass

class PayPal(PaymentProcessor):
    def process(self, amount):
        return f"Processing ${amount} via PayPal"

p = PayPal()
print(p.process(100))
  • 🔹 Function Used: `ABC`, `@abstractmethod`
  • 📘 Learned: Enforcing method implementation in subclasses
  • ♻️ Reusability: Financial frameworks, plugins
  • 💬 Comments: Ensures consistency across payment classes
8️⃣ Logging Objects with __repr__
class User:
    def __init__(self, username):
        self.username = username

    def __repr__(self):
        return f"User('{self.username}')"

u = User("sumon123")
print(repr(u))
  • 🔹 Function Used: `__repr__`
  • 📘 Learned: How to represent objects for debugging/logging
  • ♻️ Reusability: Logging, serialization, internal inspection
  • 💬 Comments: Preferred over `__str__` for dev-focused output
9️⃣ Advanced Method Overriding
class Notification:
    def send(self, message):
        return f"Sending: {message}"

class SMSNotification(Notification):
    def send(self, message):
        return f"Text: {message}"

n = SMSNotification()
print(n.send("Your code is 1234"))
  • 🔹 Function Used: Method overriding
  • 📘 Learned: Modifying parent behavior in subclass
  • ♻️ Reusability: Messaging, alert systems
  • 💬 Comments: Keeps interface same, but behavior specific
🔟 10 - Composition vs Inheritance
class Engine:
    def start(self):
        return "Engine started"

class Car:
    def __init__(self):
        self.engine = Engine()

    def drive(self):
        return f"{self.engine.start()} — Car is driving"

c = Car()
print(c.drive())
  • 🔹 Function Used: Object composition
  • 📘 Learned: Reuse through contained instances
  • ♻️ Reusability: Ideal for component-based design
  • 💬 Comments: Combines classes for flexible modeling
